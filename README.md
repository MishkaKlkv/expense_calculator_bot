# Expense Calculator Bot

Многопользовательский Telegram-бот для учета расходов. Каждый пользователь хранится отдельно по `telegram_user_id`; статистика и последние траты выбираются только для текущего Telegram-пользователя.

## Возможности MVP

- `/start` показывает главное меню.
- Добавление расхода через кнопки категорий.
- Добавление дохода через кнопки категорий.
- 21 базовая категория расходов.
- Категория `Дети`.
- Парсинг сообщений вида `овощи 500`, `овощи 500 рублей`, `coffee 10 usd`, `такси 1200`.
- Валюты: `RUB` и `USD`; если валюта не указана, используется `RUB`.
- Учет кешбека по расходам.
- Статистика расходов по категориям за текущий месяц.
- Просмотр последних трат.
- Хранение состояния диалога пользователя в PostgreSQL.
- Семейный счет: объединенная статистика и последние траты нескольких пользователей.
- Нижняя Telegram-клавиатура под полем ввода для основных действий.
- Редактирование и удаление своих трат.
- Отчеты за день, неделю, сравнение с прошлым месяцем, топ трат.
- Экспорт текущего месяца в CSV/XLSX.
- SVG-график расходов по категориям.
- Быстрый ввод без выбора категории: `продукты перекресток 580`.
- Автоопределение категорий по ключевым словам.
- Кешбек суммой или процентом, например `250` или `5%`.

## Пользовательские сценарии

### Старт

1. Пользователь отправляет `/start`.
2. Бот создает или обновляет пользователя в БД.
3. Бот показывает главное меню: добавить расход, добавить доход, статистика за месяц, последние траты.

### Добавление расхода

1. Пользователь нажимает `Добавить расход` или отправляет `/add`.
2. Бот показывает список категорий.
3. Пользователь выбирает категорию.
4. Бот переводит пользователя в состояние ожидания деталей расхода.
5. Пользователь отправляет текст покупки, например `овощи 500`.
6. Бот парсит описание, сумму и валюту. Если валюта не указана, используется `RUB`.
7. Бот спрашивает, был ли кешбек.
8. Пользователь отправляет `нет` или сумму кешбека, например `25`.
9. Бот сохраняет расход и возвращает главное меню.

### Добавление дохода

1. Пользователь нажимает `Добавить доход` или отправляет `/income`.
2. Бот показывает список категорий дохода.
3. Пользователь выбирает категорию.
4. Пользователь отправляет текст дохода, например `зарплата 150000` или `project 500 usd`.
5. Бот сохраняет доход с типом `INCOME` и возвращает главное меню.

### Статистика за месяц

1. Пользователь нажимает `Статистика за месяц` или отправляет `/stats`.
2. Бот группирует расходы текущего пользователя по категории и валюте за текущий календарный месяц.
3. Бот отправляет суммы расходов, кешбек и итог после вычета кешбека.

### Последние траты

1. Пользователь нажимает `Последние траты` или отправляет `/recent`.
2. Бот показывает последние 10 расходов текущего пользователя.

### Семейный счет

1. Первый пользователь создает семейный счет командой `/family_create`.
2. Бот выдает инвайт-код.
3. Второй пользователь отправляет `/family_join КОД`.
4. Личные расходы каждого пользователя продолжают храниться отдельно.
5. Команды `/family_stats` и `/family_recent` строят общий отчет по всем участникам семейного счета.

По умолчанию один пользователь может состоять только в одном семейном счете. Это MVP-ограничение упрощает права доступа и дальнейшее добавление ролей.

## Архитектура

- `src/bot` отвечает за Telegram UI, handlers и клавиатуры.
- `src/services` содержит бизнес-логику: парсинг, расходы, статистика, состояние диалога.
- `src/repositories` изолирует доступ к Prisma.
- `src/constants` хранит категории и валюты.
- `prisma/schema.prisma` описывает PostgreSQL-модели.

В схеме уже есть `TransactionType` с `EXPENSE` и `INCOME`; текущая модель `Expense` используется как таблица транзакций. Редактирование и удаление пока относятся к расходам.

## Запуск через Docker Compose

Этот способ поднимает все приложение целиком: Node.js бота и PostgreSQL.

1. Создайте `.env` на основе примера:

```bash
cp .env.example .env
```

2. Заполните переменные:

```env
BOT_TOKEN=telegram-bot-token
DATABASE_URL=postgresql://postgres:postgres@localhost:5432/telegram_expenses?schema=public
OPENAI_API_KEY=sk-your-openai-api-key
OPENAI_TRANSCRIPTION_MODEL=gpt-4o-mini-transcribe
NODE_ENV=development
```

Для локального запуска через `docker compose` значение `DATABASE_URL` внутри контейнера приложения будет переопределено на host `postgres`, поэтому в `.env` можно оставить localhost-строку для локальных Prisma-команд.

3. Соберите и запустите приложение:

```bash
docker compose up -d --build
```

При старте контейнер приложения выполнит:

```bash
npm run prisma:migrate:deploy
npm start
```

4. Посмотреть логи:

```bash
docker compose logs -f app
```

5. Остановить:

```bash
docker compose down
```

Остановить вместе с удалением локальной БД:

```bash
docker compose down -v
```

## Локальный запуск без Docker приложения

1. Установите зависимости:

```bash
npm install
```

2. Создайте `.env` на основе примера:

```bash
cp .env.example .env
```

3. Заполните переменные:

```env
BOT_TOKEN=telegram-bot-token
DATABASE_URL=postgresql://postgres:postgres@localhost:5432/telegram_expenses?schema=public
OPENAI_API_KEY=sk-your-openai-api-key
OPENAI_TRANSCRIPTION_MODEL=gpt-4o-mini-transcribe
NODE_ENV=development
```

4. Запустите PostgreSQL.

Через Docker:

```bash
docker compose up -d postgres
```

Если PostgreSQL установлен локально без Docker, создайте базу `telegram_expenses` вручную и убедитесь, что `DATABASE_URL` в `.env` совпадает с настройками вашего сервера.

5. Примените миграцию:

```bash
npm run prisma:migrate -- --name init
```

6. Запустите бота:

```bash
npm run dev
```

Для production-запуска:

```bash
npm start
```

## Деплой в облако

Вариант для сервисов с Docker:

1. Запушьте репозиторий на GitHub.
2. Создайте PostgreSQL в облачном сервисе.
3. Создайте app/worker из `Dockerfile`.
4. Укажите переменные окружения:

```env
BOT_TOKEN=telegram-bot-token
DATABASE_URL=postgresql://user:password@host:5432/dbname?schema=public
OPENAI_API_KEY=sk-your-openai-api-key
OPENAI_TRANSCRIPTION_MODEL=gpt-4o-mini-transcribe
NODE_ENV=production
```

5. Start command:

```bash
npm run docker:start
```

Для Railway/Render/Fly.io это должен быть always-on worker/process, потому что бот работает через Telegram polling.

## Команды бота

- `/start` - главное меню.
- `/menu` - показать главное меню с кнопками.
- `/help` - справка по командам.
- `/add` - добавить расход.
- `/income` - добавить доход.
- `/add_income` - добавить доход.
- `/edit` - редактировать свою трату.
- `/delete` - удалить свою трату.
- `/stats` - статистика за текущий месяц.
- `/recent` - последние траты.
- `/today` - расходы за сегодня.
- `/week` - расходы за неделю.
- `/compare` - сравнение с прошлым месяцем.
- `/top` - топ трат месяца.
- `/chart` - SVG-график расходов по категориям.
- `/export_csv` - экспорт CSV.
- `/export_xlsx` - экспорт XLSX.
- `/family` - информация о семейном счете.
- `/family_create` - создать семейный счет.
- `/family_join КОД` - присоединиться к семейному счету.
- `/family_stats` - семейная статистика за текущий месяц.
- `/family_recent` - последние семейные траты.
- `/family_by_user` - кто сколько потратил в семье.
