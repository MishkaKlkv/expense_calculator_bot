# Expense Calculator Bot

Многопользовательский Telegram-бот для учета расходов. Каждый пользователь хранится отдельно по `telegram_user_id`; статистика и последние траты выбираются только для текущего Telegram-пользователя.

## Возможности MVP

- `/start` показывает главное меню.
- Добавление расхода через кнопки категорий.
- Добавление дохода через кнопки категорий.
- 21 базовая категория расходов.
- Категории расходов и доходов хранятся в БД и могут настраиваться пользователем.
- Категория `Дети`.
- Парсинг сообщений вида `овощи 500`, `овощи 500 рублей`, `coffee 10 usd`, `такси 1200`.
- Валюты: `RUB` и `USD`; если валюта не указана, используется `RUB`.
- Статистика расходов и доходов по категориям за текущий месяц.
- Баланс за текущий месяц: доходы минус расходы, с рублевым итогом по текущему курсу USD.
- Счета: доступные деньги и накопления/вклады/инвестиции.
- Еженедельный отчет каждый понедельник в 10:00 по `Europe/Moscow`.
- Плановые ежемесячные платежи.
- Напоминания о плановых платежах в день оплаты.
- Просмотр последних трат.
- Хранение состояния диалога пользователя в PostgreSQL.
- Семейный счет: объединенная статистика и последние траты нескольких пользователей.
- Нижняя Telegram-клавиатура под полем ввода для основных действий.
- Подсказки команд при вводе `/` и кнопка меню команд Telegram.
- Короткая `/help`: подробные действия вынесены в кнопки внутри разделов.
- Геймификация регулярного учета: серия дней, XP, 30 уровней, достижения и `/done` для дней без расходов.
- Редактирование и удаление своих трат.
- Редактирование и удаление своих доходов.
- Очистка своих расходов за текущий месяц или за все время с подтверждением.
- Отчеты за день, неделю, сравнение с прошлым месяцем, топ трат.
- Экспорт текущего месяца в CSV/XLSX.
- SVG-график расходов по категориям.
- Быстрый ввод без выбора категории: `продукты перекресток 580`.
- Автоопределение категорий по ключевым словам.
- Кешбек учитывается как отдельный доход в категории `Кешбек`.
- Отмена текущего диалога через `/cancel`.
- Ежедневное напоминание внести траты за день в выбранное время.

## Пользовательские сценарии

### Старт

1. Пользователь отправляет `/start`.
2. Бот создает или обновляет пользователя в БД.
3. Бот показывает главное меню: добавить расход, добавить доход, статистика за месяц, последние траты.

### Добавление расхода

1. Пользователь нажимает `Добавить расход` или отправляет `/add`.
2. Бот показывает список категорий.
3. Пользователь выбирает категорию.
4. Бот переводит пользователя в состояние ожидания деталей расхода.
5. Пользователь отправляет текст покупки, например `овощи 500`.
6. Бот парсит описание, сумму и валюту. Если валюта не указана, используется `RUB`.
7. Бот сохраняет расход и возвращает главное меню.
8. Первый учтенный день за сутки дает XP и поддерживает серию учета.

### Геймификация учета

1. Пользователь получает XP за первый расход или доход в день.
2. Если расходов не было, пользователь отправляет `/done`, чтобы честно отметить день без расходов.
3. `/done вчера` можно использовать, если пользователь вспомнил про учет на следующий день.
4. `/profile` показывает уровень, XP, серию и достижения.
5. `/streak` показывает текущую и лучшую серию учета.

### Добавление дохода

1. Пользователь нажимает `Добавить доход` или отправляет `/add_income`.
2. Бот показывает список категорий дохода.
3. Пользователь выбирает категорию.
4. Пользователь отправляет текст дохода, например `зарплата 150000` или `project 500 usd`.
5. Бот сохраняет доход с типом `INCOME` и возвращает главное меню.

Кешбек добавляется как доход в категории `Кешбек`, например `кешбек 250`. Такие доходы можно вводить несколько раз за месяц.

### Статистика за месяц

1. Пользователь нажимает `Статистика за месяц` или отправляет `/stats`.
2. Бот группирует расходы и доходы текущего пользователя по категории и валюте за текущий календарный месяц.
3. Бот отправляет суммы расходов и доходов по категориям.

### Баланс за месяц

1. Пользователь отправляет `/balance`.
2. Бот считает доходы, расходы и разницу по каждой валюте.
3. Если есть USD, бот конвертирует доходы и баланс в RUB по текущему курсу ЦБ РФ.
4. Бот показывает блок `Деньги сейчас`: всего, накоплено, доступно.

### Счета

1. Пользователь добавляет счет командой `/account_add Карта | доступно | 150000`.
2. Для накоплений используется тип `накопления`, например `/account_add Вклад | накопления | 200000`.
3. Пользователь смотрит сводку через `/accounts`, меняет сумму через `/account_set ID 175000` и удаляет счет через `/account_delete ID`.

### Ежедневное напоминание

1. Пользователь отправляет `/reminder 20:00`.
2. Бот сохраняет время и каждый день присылает напоминание внести траты.
3. Пользователь может проверить статус через `/reminder`.
4. Пользователь может выключить напоминание через `/reminder_off` и включить обратно через `/reminder_on`.

### Еженедельный отчет

1. Каждый понедельник в 10:00 по `Europe/Moscow` бот отправляет отчет за прошлую неделю.
2. Пользователь может запросить такой отчет вручную через `/weekly_report`.
3. В отчете есть расходы, доходы, крупные расходы и плановые платежи на текущую неделю.

### Плановые платежи

1. Пользователь добавляет ежемесячный платеж командой `/planned_add 5 | Интернет | Домашний интернет | 700`.
2. Бот сохраняет день месяца, категорию, описание, сумму и валюту.
3. Бот предлагает включить напоминание в день платежа в 10:00.
4. Пользователь смотрит список через `/planned` и удаляет платеж через `/planned_delete ID`.

### Последние траты

1. Пользователь нажимает `Последние траты` или отправляет `/recent`.
2. Бот показывает последние 10 расходов текущего пользователя.

### Семейный счет

1. Первый пользователь создает семейный счет командой `/family_create`.
2. Бот выдает инвайт-код.
3. Второй пользователь отправляет `/family_join КОД`.
4. Личные расходы каждого пользователя продолжают храниться отдельно.
5. Команды `/family_stats` и `/family_recent` строят общий отчет по всем участникам семейного счета.
6. Владелец может переименовать счет через `/family_rename НАЗВАНИЕ`.
7. Владелец может удалить участника кнопкой в `/family`.

По умолчанию один пользователь может состоять только в одном семейном счете. Это MVP-ограничение упрощает права доступа и дальнейшее добавление ролей.

## Архитектура

- `src/bot` отвечает за Telegram UI, handlers и клавиатуры.
- `src/services` содержит бизнес-логику: парсинг, расходы, статистика, состояние диалога.
- `src/repositories` изолирует доступ к Prisma.
- `src/constants` хранит категории и валюты.
- `prisma/schema.prisma` описывает PostgreSQL-модели.

В схеме уже есть `TransactionType` с `EXPENSE` и `INCOME`; текущая модель `Expense` используется как таблица транзакций. Редактирование и удаление пока относятся к расходам.

## Запуск через Docker Compose

Этот способ поднимает все приложение целиком: Node.js бота и PostgreSQL.

1. Создайте `.env` на основе примера:

```bash
cp .env.example .env
```

2. Заполните переменные:

```env
BOT_TOKEN=telegram-bot-token
DATABASE_URL=postgresql://postgres:postgres@localhost:5432/telegram_expenses?schema=public
OPENAI_API_KEY=sk-your-openai-api-key
OPENAI_TRANSCRIPTION_MODEL=gpt-4o-mini-transcribe
ADMIN_TELEGRAM_IDS=254527616
NODE_ENV=development
```

Для локального запуска через `docker compose` значение `DATABASE_URL` внутри контейнера приложения будет переопределено на host `postgres`, поэтому в `.env` можно оставить localhost-строку для локальных Prisma-команд.

3. Соберите и запустите приложение:

```bash
docker compose up -d --build
```

При старте контейнер приложения выполнит:

```bash
npm run prisma:migrate:deploy
npm start
```

4. Посмотреть логи:

```bash
docker compose logs -f app
```

5. Остановить:

```bash
docker compose down
```

Остановить вместе с удалением локальной БД:

```bash
docker compose down -v
```

## Локальный запуск без Docker приложения

1. Установите зависимости:

```bash
npm install
```

2. Создайте `.env` на основе примера:

```bash
cp .env.example .env
```

3. Заполните переменные:

```env
BOT_TOKEN=telegram-bot-token
DATABASE_URL=postgresql://postgres:postgres@localhost:5432/telegram_expenses?schema=public
OPENAI_API_KEY=sk-your-openai-api-key
OPENAI_TRANSCRIPTION_MODEL=gpt-4o-mini-transcribe
ADMIN_TELEGRAM_IDS=254527616
NODE_ENV=development
```

4. Запустите PostgreSQL.

Через Docker:

```bash
docker compose up -d postgres
```

Если PostgreSQL установлен локально без Docker, создайте базу `telegram_expenses` вручную и убедитесь, что `DATABASE_URL` в `.env` совпадает с настройками вашего сервера.

5. Примените миграцию:

```bash
npm run prisma:migrate -- --name init
```

6. Запустите бота:

```bash
npm run dev
```

Для production-запуска:

```bash
npm start
```

## Деплой в облако

Вариант для сервисов с Docker:

1. Запушьте репозиторий на GitHub.
2. Создайте PostgreSQL в облачном сервисе.
3. Создайте app/worker из `Dockerfile`.
4. Укажите переменные окружения:

```env
BOT_TOKEN=telegram-bot-token
DATABASE_URL=postgresql://user:password@host:5432/dbname?schema=public
OPENAI_API_KEY=sk-your-openai-api-key
OPENAI_TRANSCRIPTION_MODEL=gpt-4o-mini-transcribe
ADMIN_TELEGRAM_IDS=254527616
NODE_ENV=production
```

5. Start command:

```bash
npm run docker:start
```

Для Railway/Render/Fly.io это должен быть always-on worker/process, потому что бот работает через Telegram polling.

## Команды бота

- `/start` - главное меню.
- `/menu` - показать главное меню с кнопками.
- `/help` - справка по командам.
- `/cancel` - отменить текущее действие.
- `/profile` - профиль учета, уровень и достижения.
- `/streak` - серия учета.
- `/done` - отметить день без расходов.
- `/add` - добавить расход.
- `/balance` - баланс за текущий месяц.
- `/income` - доходы за текущий месяц.
- `/add_income` - добавить доход.
- `/accounts` - счета, накопления и доступные деньги.
- `/account_add Карта | доступно | 150000` - добавить счет.
- `/account_set ID 175000` - изменить сумму счета.
- `/account_delete ID` - удалить счет.
- `/reminder 20:00` - включить ежедневное напоминание.
- `/reminder` - статус ежедневного напоминания.
- `/reminder_off` - выключить ежедневное напоминание.
- `/reminder_on` - включить ежедневное напоминание снова.
- `/reminder_test` - проверить текст напоминания, доступно только админам.
- `/reminder_check` - проверить расписание напоминания, доступно только админам.
- `/reminder_run` - вручную запустить проверку и отправку due-напоминаний, доступно только админам.
- `/admin_stats` - статистика использования бота, доступна только Telegram ID из `ADMIN_TELEGRAM_IDS`.
- `/edit` - редактировать свою операцию.
- `/delete` - удалить свою операцию.
- `/clear_month_expenses` - очистить расходы за текущий месяц.
- `/clear_all_expenses` - очистить все расходы.
- `/stats` - статистика за текущий месяц.
- `/prev_month` - статистика за прошлый месяц.
- `/weekly_report` - отчет за прошлую неделю.
- `/planned` - список плановых платежей.
- `/planned_add 5 | Интернет | Домашний интернет | 700` - добавить ежемесячный плановый платеж.
- `/planned_reminder_on ID` - включить напоминание о плановом платеже.
- `/planned_reminder_off ID` - выключить напоминание о плановом платеже.
- `/planned_delete ID` - удалить плановый платеж.
- `/recent` - последние траты.
- `/today` - расходы за сегодня.
- `/week` - расходы за неделю.
- `/compare` - сравнение с прошлым месяцем.
- `/top` - топ трат месяца.
- `/chart` - SVG-график расходов по категориям.
- `/export_csv` - экспорт CSV.
- `/export_xlsx` - экспорт XLSX.
- `/categories` - список категорий.
- `/category_add расход НАЗВАНИЕ` - добавить категорию расходов.
- `/category_add доход НАЗВАНИЕ` - добавить категорию доходов.
- `/category_delete расход НАЗВАНИЕ` - удалить категорию расходов.
- `/category_delete доход НАЗВАНИЕ` - удалить категорию доходов.
- `/family` - информация о семейном счете.
- `/family_create` - создать семейный счет.
- `/family_join КОД` - присоединиться к семейному счету.
- `/family_rename НАЗВАНИЕ` - переименовать семейный счет.
- `/family_stats` - семейная статистика за текущий месяц.
- `/family_recent` - последние семейные траты.
- `/family_by_user` - кто сколько потратил в семье.
